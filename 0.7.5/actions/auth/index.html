Version: 0.7.5

AUTH

Change global authn/authz settings

LOGIN​

About

Time complexity: O(1)
Accept type:

  • AnyArray

Return type:

  • Rcode 0
  • Rcode 10

Syntax:

  • AUTH LOGIN <username> <token>

Attempts to log in using the provided credentials

CLAIM​

About

Time complexity: O(1)
Accept type:

  • AnyArray

Return type:

  • String
  • Rcode 10

Syntax:

  • AUTH CLAIM <origin key>

Attempts to claim the root account using the origin key

LOGOUT​

About

Time complexity: O(1)
Accept type:

  • AnyArray

Return type:

  • Rcode 0
  • Rcode 10

Syntax:

  • AUTH LOGOUT

Attempts to log out the currently logged in user

ADDUSER​

About

Time complexity: O(1)
Accept type:

  • AnyArray

Return type:

  • String
  • Rcode 11

Syntax:

  • AUTH ADDUSER <username>

Attempts to create a new user with the provided username, returning the token

DELUSER​

About

Time complexity: O(1)
Accept type:

  • AnyArray

Return type:

  • Rcode 0
  • Rcode 10
  • Rcode 11

Syntax:

  • AUTH DELUSER <username>

Attempts to delete the user with the provided username

RESTORE​

About

Time complexity: O(1)
Accept type:

  • AnyArray

Return type:

  • String
  • Rcode 10
  • Rcode 11

Syntax:

  • AUTH RESTORE <username>
  • AUTH RESTORE <origin-key> <username>

Attempts to restore the password for the provided user. This will regenerate the token and return the newly issued token. However, if you aren't a root account, that is, you lost your root password, then you'll need to run AUTH RESTORE <origin-key> root.

LISTUSER​

About

Time complexity: O(1)
Accept type:

  • AnyArray

Return type:

  • Non-null array

Syntax:

  • AUTH LISTUSER

Attempts to return a list of users for the current database instance

WHOAMI​

About

Time complexity: O(1)
Accept type:

  • AnyArray

Return type:

  • String

Syntax:

  • AUTH WHOAMI

Returns a string with the AuthID of the currently logged in user or errors if the user is not logged in
